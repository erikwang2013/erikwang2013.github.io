Cloudflare 页面

Cloudflare 是一家云解决方案提供商，拥有庞大的专有内容交付网络 (CDN)。与 Netlify 或 Vercel 一样，Cloudflare Pages 使部署过程灵活而简单。您可以将 GitHub 存储库添加到该服务，并在每次 PR 后自动构建和托管基于 Zola 的网站。

🔗循序渐进

  1. 登录或创建一个新的 Cloudflare 帐户，然后在右侧导航栏中选择“页面”
  2. 按“创建项目”按钮
  3. 选择包含您的 Zola 网站的 GitHub 存储库并将其连接到 Cloudflare Pages
  4. 点击“开始设置”
  5. 输入您的项目名称。请记住，如果您想使用默认页面域 (pages.dev)，这将是您网站未来的 URL（“yourprojectname.pages.dev”）。此外，选择一个生产分支。
  6. 在构建设置中，选择 Zola 作为框架预设。构建命令和构建输出目录将自动填充。
  7. 切换下面的环境变量并添加ZOLA_VERSION为变量名称。使用0.13.0或不同的 Zola 版本作为值。
  8. 最后，保存并部署。

您的网站现已构建并部署到 Cloudflare 的网络！您可以在页面仪表板中添加自定义域或修改设置。

您可以在开发者门户中找到文档和指南，例如Cloudflare Pages 入门和 使用 Cloudflare Pages 部署 Zola 。
